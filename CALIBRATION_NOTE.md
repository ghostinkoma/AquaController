# キャリブレーションノート — AquaController センサ独り立ち

高価な基準センサ **SHT31 / BME680** を校正完了後に取り外し、常用センサ **AHT25 / BMP280**
単独（＝独り立ち）で運用するための実務メモ。README の「自動校正」節を実機検証で補足するもの。

> 数値はすべて**実機（東芝 C7-5020L-07 ファン搭載機, 2026-07 検証）の実測**に基づく。
> 個体・設置で変わるため、必ず自分の機体で `⚙システムタブ → 校正カード` / `GET /api/state` の
> `calib` を確認すること。本ノートの具体値は「桁感」の参考。

---

## 1. 何を学習するか（差分＝「基準 − 作業」）

| 項目 | 作業センサ | 基準センサ | 適用先 (`g_calib`) |
|---|---|---|---|
| 気温 | AHT25 | **SHT31 単独** | `air` → AHT 温度に加算 |
| 湿度 | AHT25 | **SHT31 単独** | `humid` → AHT 湿度に加算 |
| 気圧 | BMP280 | **BME680** | `press` → BMP280 気圧に加算 |

- **温度基準は SHT31 のみ**。BME680 はガスヒータの自己発熱で温度が +0.5〜1℃ 高く出るため
  温度基準に不適 → BME680 は**気圧基準専用**。SHT31 が無い時のみ BME680 温湿へフォールバック。
- 気圧表示: 基準 BME680 があれば基準を無補正表示、**無ければ BMP280 + `g_calib.press`（独り立ち）**。
  基準を外した瞬間に自動で切り替わる（`sensors.cpp readAir`）。

---

## 2. 最重要ノウハウ：**気流ゲート**（勾配汚染の排除）

**問題**: 静止空気では、作業センサと基準センサが**設置位置の熱勾配**で乖離する。実測では
ファン停止時 diffAir=**+1.3℃**、ファン運転時 **−0.4℃** と、同じ2センサでも **±1℃以上変動**した。
この状態で校正すると、勾配（設置の癖）を「センサ誤差」と誤認して offset を汚染する
（実際に一度 air offset が誤って +0.65℃ になった）。

**原理**: **強い気流を両センサに当てると、強制対流で両者が同じ「流れる空気温度」に収束**する。
数十秒でローカル勾配が消え、残差＝**純粋なセンサ固有誤差**になる。実測（ファン70%, 強気流下）:

| 項目 | 純粋オフセット | σ | 評価 |
|---|---|---|---|
| 温度 diffAir | **≈ 0.0〜0.2℃** | 0.01〜0.03 | ほぼゼロ（AHT25 は優秀） |
| 気圧 diffPress | **≈ −0.06〜−0.08 hPa** | 0.03 | ほぼゼロ（BMP280 は良個体） |
| 湿度 diffHumid | **≈ +2.3〜2.7 %RH** | 0.1〜1.0 | 実在するセンサ差 |

**実装**: 自動校正は **ファン duty ≥ `calibauto::MIN_DUTY_FOR_CALIB`(40%) の時だけ**サンプルを採る
（`sensors.cpp calibReadOnce` 冒頭で `g_live.fanDuty` を判定）。これで静止空気の汚染を自動排除し、
offset が純粋値へ収束する。**校正したい時はファンを回す**（自然な冷却運転中でも可）。

> 6.7時間の実機ログで、手動設定 air=0.0 → 自動校正が **+0.197℃**（気流下 diffAir と一致）へ
> 精緻化し、以後 σ0.03 で安定。温度残差 <0.2℃ で目標 0.5℃ をクリアした。

## 3. 独り立ち手順（時間プラン）

**目標許容誤差: 気温 ±0.5℃ / 気圧 ±2 hPa**（湿度は参考）。

1. **校正運転（約 1 日）**: 基準を接続したまま通常運転。ファンが回る時間帯に気流ゲート校正が働く。
   温度の日内変動を跨ぐと条件に依らない offset が得られる。最低数時間、推奨 24 時間。
2. **収束確認**（校正カード or `GET /api/state.calib`）:
   - 「適用オフセット」と「ライブ差分（基準−作業）」を比較。**残差 = |ライブ差分 − オフセット|** が
     許容内（気温<0.5℃, 気圧<2hPa）で安定していれば合格。強気流下で確認するのが最も確実。
   - `calib.json` / CSV をバックアップ（校正カードのボタン）。
3. **手動即時セット（任意・確実）**: `POST /api/calib {"air":0.0,"press":-0.07,"humid":2.7}` で
   実測純粋値を直接書込（CRC付き永続化）。`{"reset":true}` で既定(0)へ戻せる。
4. **取り外し**: 電源を切り SHT31 / BME680 を外す（12V 混在環境の事故防止・高価パーツ回収）。
   → 気圧表示は自動で「BMP280 + オフセット」へ、温湿は AHT25 + オフセットへ。`calib.json` は保持。
5. **凍結（任意）**: 再校正しないなら `calibauto::ENABLE=false` にして再ビルド + 書込。
   基準が無ければ校正タスクは自動起動しないので必須ではない（`ENABLE=true` のままでも独り立ち動作）。

## 4. 落とし穴・実機知見

- **BMP280 クローン**: 補正係数不整合で気圧がガベージ（>1085/<800 hPa）になる個体は独り立ち不可
  → ライブ差分が不安定/棄却される。正規 BMP280 か BME280 へ交換。本機の個体は −0.08hPa と優秀だった。
- **ファン気流に晒される位置**: 空気センサがファン直下だと、ファン ON/OFF で読みが ±1℃ 動く。
  気流ゲート校正はこれを利用して純粋値を取るが、**表示の絶対値**は気流状態に依存する点は理解しておく。
- **手動 offset は気流ゲート自動校正に上書きされる**: 基準を残したまま `POST /api/calib` で入れた値は、
  ファンが回ると自動校正が同等の純粋値へ更新する（同値なので実害なし）。基準を外せば凍結。
- **書込は USB 推奨（当面）**: 大容量 WiFi OTA が exit56 で不安定な環境があった（本体・無線の別課題）。
  独り立ち確定（`ENABLE=false`）の書込は `pio run -e esp32-c3 -t upload`（COM ポート）で確実。

## 5. 関連 API / 設定

- `GET  /api/state` … `calib{air,press,humid, diffAir,diffHumid,diffPress,diffValid,diffPressValid}`
- `POST /api/calib` … `{air,press,humid,water}` 手動設定 / `{"reset":true}` 既定化（CRC付き保存）
- `config.h`: `calibauto::{ENABLE, MIN_DUTY_FOR_CALIB, SHT31_ADDR, BME680_ADDR, SAMPLE_PERIOD_MS,
  SAMPLES_PER_WRITE}` / `calib::{WATER,AIR,PRESS,HUMID}_OFFSET_*`（起動既定）
- 永続: `/calib.json`（CRC + `.old` フェイルセーフ）
